# Feature-005 项目管理页面能力规范

## 1. 文档定位
- 类型：能力基线文档（按新需求模板编排）
- 页面：`/projects`
- 目标：沉淀当前已实现的项目管理能力，作为后续迭代需求输入

## 2. 背景与目标
项目管理页承担“项目资产管理 + 仓库信息同步”的入口职责。当前能力目标：
1. 提供项目列表查看与状态筛选。
2. 支持新建项目，建立后续任务上下文。
3. 支持修改项目基础信息。
4. 支持删除项目。
5. 支持仓库信息同步，回填技术栈等元数据。

## 3. 用户故事
1. 作为平台用户，我希望查看项目清单及状态，快速定位可用项目。
2. 作为配置人员，我希望创建项目并填写仓库地址与默认分支。
3. 作为管理员，我希望删除误建项目。
4. 作为运维/研发人员，我希望手动同步项目仓库信息以更新技术栈。
5. 作为项目管理员，我希望在列表中直接修改项目配置（展示名、仓库、分支、描述、状态），无需重建项目。

## 4. 功能范围（当前）
1. 项目列表（ProTable）
   - 字段：显示名称、标识代码、代码库地址、技术栈、分支、状态、最后同步时间、操作。
   - 状态筛选：`active` / `archived`。
2. 新建项目（ModalForm）
   - 字段：`name`、`display_name`（必填），`repo_url`、`branch`（默认 `main`）、`description`。
3. 删除项目
   - 行级删除按钮，带二次确认。
4. 修改项目
   - 行级“编辑”入口，打开编辑表单。
   - 支持更新：`display_name`、`repo_url`、`branch`、`description`、`status`。
   - 提交调用 `PUT /api/v1/projects/{project_id}`。
5. 同步仓库信息
   - 行级同步按钮，仅 `repo_url` 存在时可用。
   - 同步成功提示技术栈，失败提示错误原因。

## 5. 非目标
- 不提供项目详情页跳转。
- 不提供批量同步/批量删除。

## 6. 验收标准
1. 页面可分页加载项目列表，支持状态筛选。
2. 新建项目成功后列表刷新。
3. 编辑项目成功后列表刷新，且仅更新提交字段。
4. 删除项目成功后列表刷新。
5. 同步成功后展示 `tech_stack` 提示并刷新列表。
6. 同步按钮在无 `repo_url` 时禁用。
