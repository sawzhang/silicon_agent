2. 六层上下文保障机制
Layer 1
prior_outputs 累积传递
每个 stage 完成后，原始输出追加到 prior_outputs[] 列表。 下一 stage 通过 build_user_prompt() 读取所有已完成阶段的输出并注入 prompt。
关键设计：Agent 间无共享会话 — 每次 runner.chat(prompt, reset=True) 清空对话历史，所有跨阶段上下文靠 prompt 显式传递。
Layer 2
滑动窗口压缩
按距离分级压缩：紧邻阶段看全文 (L2)，隔一阶段看摘要 (L1)，更远看一句话 (L0)。 模板可用 context_from 强制指定全文传递。
CompressionResult.build_prior_context(stage_index) — 自动计算距离并选择压缩级别，平衡上下文完整性与 token 消耗。
Layer 3
结构化产出提取
每个 stage 完成后，LLM 提取结构化 JSON（如 files_modified, tests_passed）。 驱动条件跳过、动态门控和路由决策。
extract_structured_output() → stage.output_structured (DB JSON)。 结构化数据同时丰富 L0/L1 压缩摘要。
Layer 4
项目级跨任务记忆
文件存储在 memory/<project_id>/，分 conventions / architecture / patterns / issues 四类，按角色过滤加载。
任务完成后 LLM 从所有 stage 输出提取可复用知识写入记忆。 每个 stage 执行前加载最近 10 条相关记忆注入 prompt。
Layer 5
失败重试上下文 (Ralph Loop V2)
stage 失败重试时，注入根因分析、教训、建议和上次部分输出（截断 2000 字符）。 教训同步写入项目记忆 issues 类别。
若 SKILL_REFLECTION_ENABLED，调用 generate_structured_reflection() 进行 LLM 结构化反思，而非简单拼接错误信息。
Layer 6
Gate 审批反馈回注
人工审批 Gate 被拒绝时，审批者 comment 作为 gate_rejection_context 注入重新执行的 stage prompt 中。
格式：## Gate审批被拒绝 + 重试次数 + 审批者反馈。 最多重试 3 次，每次都携带最新反馈。
